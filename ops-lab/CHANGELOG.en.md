# Changes And Research Direction

[한국어](./CHANGELOG.md)

This document summarizes service-level change context and current research
direction for my personal operations workspace at a public-safe level. It is not
a detailed changelog of private repositories or server configuration.

## Vodex

- Documented the role of the private AI workspace and Codex-compatible adapter at a public-safe level.
- Current direction: improving practical AI support for server operations, documentation, code review, and local-context workflow automation.

## Vodex Console

- Framed it as a separate operator-console experiment rather than a small helper utility.
- Current direction: session persistence, streaming state, process control, and a more direct AI work interface for server tasks.

## Career AI

- Documented it only at the workflow level to avoid exposing private career materials.
- Current direction: structured, evidence-based career positioning, role analysis, and application-context management.

## Internal Lounge Platform

- Grouped user screens, admin screens, and shared packages into one internal operations platform context.
- Current direction: shared domain types, clear deployment boundaries, and separation between user-facing and admin workflows.

## Admin Dashboard

- Framed it as a product operations system for operational data, reports, permissions, exports, and storage workflows.
- Current direction: dense admin UX and data-access patterns that reduce operating cost.

## User Workflow App

- Described it as a product for real work processes rather than a feature list.
- Current direction: realtime operations, permissions, mobile usability, and incremental feature expansion.

## Knowledge Vault

- Framed the private knowledge store as infrastructure for leadership notes, technical decisions, and project history.
- Current direction: making review culture, decisions, and operating notes reusable as context for AI-assisted work.

## Daily Writing

- Framed the local writing assistant as a human-reviewed AI workflow.
- Current direction: draft generation, draft preservation, editing support, and avoiding automatic publishing.

## Local DNS

- Framed local DNS support as a small operations tool for improving service access.
- Current direction: predictable LAN service access without exposing internal routing details.

## Public Profile

- Shifted the public profile from server topology toward operated services and engineering ownership.
- Current direction: showing career positioning and service context without exposing private server details.

## Quartz Publishing

- Framed it as a publishing layer for selected content from a private knowledge base.
- Current direction: maintaining a repeatable flow from private notes to public-safe pages.

## Lightweight Web Services

- Framed small static/PHP utilities as lightweight experiments separate from the main profile.
- Current direction: preserving useful prototypes without diluting the core message of the public profile.

## Obsidian Sync

- Documented the sync layer only at the capability level, without accounts, paths, or access details.
- Current direction: controlled synchronization and automation access for a private knowledge store.

## Ops Lab Documentation

- Organized service contexts and public-safe change direction inside `ops-lab`.
- Current direction: keeping the context of real operated services updated while separating secrets and internal operations data from public docs.
