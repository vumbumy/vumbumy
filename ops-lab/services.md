# 서비스 컨텍스트

[English](./services.en.md)

이 문서는 개인 서버 워크스페이스에서 운영하거나 연구 중인 서비스들을
공개 가능한 수준으로 정리합니다. 비공개 설정이 아니라, 제품 방향,
엔지니어링 문제, 운영에서 얻는 학습에 집중합니다.

## 현재 연구 방향

활발한 작업은 세 가지 주제로 수렴하고 있습니다.

- AI 보조 운영과 개발 워크플로우
- 개인 지식과 커리어 맥락 관리
- 작은 프로덕션 환경 운영: 라우팅, 헬스, 지속성, 복구

## 운영 및 연구 중인 서비스

| 서비스 | 공개 가능한 맥락 | 현재 방향 |
| --- | --- | --- |
| `vodex` | LibreChat과 Codex 호환 어댑터를 중심으로 만든 사설 AI 작업 환경입니다. | 서버 운영, 문서화, 코드 리뷰, 워크플로우 자동화에 AI를 붙이되 로컬 컨텍스트 접근을 통제하는 방향을 연구합니다. |
| `vodex-console` | 개인 서버에서 Codex 세션을 더 직접적으로 다루기 위한 운영자 콘솔 실험입니다. | 세션 지속성, 스트리밍 상태 표시, 프로세스 제어, 서버 작업에 맞는 인터페이스를 실험합니다. |
| `ops-toolbox` | 서버 운영 중 필요한 Git, 네트워크, 데이터베이스, Docker 보조 도구를 모아둔 관리용 컨테이너입니다. | 운영 환경마다 도구가 부족해 작업이 끊기는 문제를 줄이고, 점검과 복구 작업을 재현 가능한 쉘 환경으로 모읍니다. |
| `career-ai` | 역할 맥락, 포지셔닝, 지원 자료를 정리하는 개인 커리어 워크플로우입니다. | 비공개 이력서나 지원 데이터는 공개하지 않고, 구조화된 근거 기반으로 커리어 판단을 돕는 흐름을 다듬습니다. |
| `obsidian-vault` | 리더십 노트, 기술적 결정, 프로젝트 히스토리를 구조화하는 비공개 지식 베이스입니다. | 흩어진 노트를 리뷰, 의사결정, AI 보조 작업에 재사용 가능한 운영 컨텍스트로 바꿉니다. |
| `daily-writing` | 짧은 글 후보를 생성, 저장, 수정, 개선하는 로컬 글쓰기 보조 도구입니다. | 자동 게시가 아니라 사람이 검토하는 AI 보조 콘텐츠 워크플로우를 실험합니다. |
| `beszel` | 개인 서버 상태를 가볍게 관찰하기 위한 모니터링 허브입니다. | 대형 관제보다 디스크, 컨테이너, 자원 사용량 같은 운영 신호를 빠르게 확인하는 데 집중합니다. |
| `security-report` | 서버 보안 신호를 정리해 알려주는 작은 자동 리포트입니다. | 로그인 실패, 중요 컨테이너 상태, 디스크 압박, 외부 노출처럼 놓치기 쉬운 항목만 추려 확인하는 방향입니다. |
| `local-dns` | LAN 안에서 개인 도메인 접근을 안정화하기 위한 작은 DNS 보조 서비스입니다. | 내부 라우팅 세부사항을 공개하지 않으면서 로컬 서비스 접근성을 일정하게 유지합니다. |
| [`index-page`](https://vumy.kr/) | 공개 프로필과 서비스 인덱스입니다. | 원시 서버 구조가 아니라 직접 운영하는 서비스와 엔지니어링 오너십이 보이도록 다듬습니다. |
| `quartz-obsidian` | 선택한 지식 베이스 내용을 퍼블리싱하기 위한 Quartz 기반 레이어입니다. | 비공개 노트에서 공개 가능한 페이지로 이어지는 반복 가능한 경로를 유지합니다. |
| `vumy_kr` | 개인 웹 워크스페이스에 남아 있는 작은 PHP 기반 사이트/서비스입니다. | 레거시 또는 가벼운 웹 실험을 메인 프로필과 섞지 않고 관리합니다. |
| `sudoku-solver` | nginx로 서빙되는 작은 정적 유틸리티입니다. | 간단한 알고리즘 유틸리티를 가벼운 웹 서비스 형태로 유지합니다. |
| `obsidian-sync` | 비공개 지식 저장소를 동기화하기 위한 WebDAV 레이어입니다. | 계정, 경로, 접근 정보를 공개하지 않으면서 노트 동기화와 자동화 접근을 지원합니다. |
| `career-vault` | 비공개 커리어 워크스페이스의 호환성 진입점입니다. | 공개 문서에서는 `career-ai` 맥락으로 통합해서 다룹니다. |
| `immich-migration` | 사진 서버를 더 안정적인 주 서버 구조로 옮기기 위한 운영 작업입니다. | 기존 미디어를 보존하면서 새 저장소, 읽기 전용 레거시 소스, 복구 절차를 함께 정리합니다. |
| `immich-dedup` | 사진 중복을 다룰 때 원본 보존과 업로드 안정성을 함께 고려하는 보조 작업입니다. | 무조건 삭제보다 휴지통, 매니페스트, 품질 비교, 모바일 백업 안정성을 우선하는 정책을 실험합니다. |

## 포트폴리오 관점의 신호

이 서비스들은 “컨테이너를 많이 띄운다”는 것을 보여주기 위한 목록이
아닙니다. 반복되는 일을 유지보수 가능한 시스템으로 바꿔가는 과정이
핵심입니다.

- 개인 AI 운영은 일회성 프롬프트가 아니라 통제된 워크플로우로 정리되고 있습니다.
- 지식과 커리어 자료는 구조화된 컨텍스트로 관리해 AI 보조 작업의 신뢰도와 감사 가능성을 높입니다.
- 작은 유틸리티도 서비스 경계, 지속성, 문서화를 고려해 운영합니다.
- 사진 서버처럼 개인 데이터가 걸린 서비스는 기능보다 복구 가능성과 보존 정책을 먼저 봅니다.

## 공개 범위

이 문서는 비공개 설정, 내부 URL, 시크릿, 서버 경로, 데이터베이스 인증
정보, 미디어 위치, 사용자 데이터, 사내 세부 정보를 공개하지 않습니다.
