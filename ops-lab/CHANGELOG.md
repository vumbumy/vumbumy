# 변경 및 연구 방향

[English](./CHANGELOG.en.md)

이 문서는 개인 운영 워크스페이스의 서비스별 변경 맥락과 현재 연구
방향을 공개 가능한 수준으로 정리합니다. 비공개 저장소나 서버 설정의
상세 변경 로그가 아니라, 외부에서 읽어도 안전한 운영 포트폴리오 기록입니다.

## Vodex

- 사설 AI 작업 환경과 Codex 호환 어댑터의 역할을 공개 가능한 수준으로 정리했습니다.
- 현재 방향: 서버 운영, 문서화, 코드 리뷰, 로컬 컨텍스트 기반 워크플로우 자동화에 AI를 실용적으로 붙이는 방식 개선.

## Vodex Console

- 단순 보조 유틸리티가 아니라 별도의 운영자 콘솔 실험으로 정리했습니다.
- 현재 방향: 세션 지속성, 스트리밍 상태, 프로세스 제어, 서버 작업에 맞는 더 직접적인 AI 작업 인터페이스.

## Career AI

- 비공개 커리어 자료가 노출되지 않도록 워크플로우 수준으로만 문서화했습니다.
- 현재 방향: 구조화된 근거 기반의 커리어 포지셔닝, 역할 분석, 지원 맥락 관리.

## Internal Lounge Platform

- 사용자 화면, 관리자 화면, 공유 패키지의 방향을 하나의 내부 운영 플랫폼 맥락으로 묶었습니다.
- 현재 방향: 공통 도메인 타입, 명확한 배포 경계, 사용자-facing 워크플로우와 관리자 워크플로우의 분리.

## Admin Dashboard

- 운영 데이터, 리포트, 권한, 내보내기, 저장소 워크플로우를 다루는 제품 운영 시스템으로 정리했습니다.
- 현재 방향: 정보 밀도가 높은 관리자 UX와 비용을 줄이는 데이터 접근 패턴.

## User Workflow App

- 기능 목록이 아니라 실제 업무 흐름을 다루는 제품으로 설명했습니다.
- 현재 방향: 실시간 운영, 권한, 모바일 사용성, 기능 단위의 점진적 확장.

## Knowledge Vault

- 비공개 지식 저장소를 리더십 노트, 기술 결정, 프로젝트 히스토리 인프라로 정리했습니다.
- 현재 방향: 리뷰 문화, 의사결정, 운영 노트를 AI 보조 작업에 재사용 가능한 컨텍스트로 만드는 것.

## Daily Writing

- 로컬 글쓰기 보조 도구를 사람이 검토하는 AI 워크플로우로 정리했습니다.
- 현재 방향: 글 후보 생성, 초안 보존, 편집 지원, 자동 게시 방지.

## Local DNS

- 로컬 DNS 보조 서비스를 서비스 접근성을 높이는 작은 운영 도구로 정리했습니다.
- 현재 방향: 내부 라우팅 세부사항을 공개하지 않으면서 LAN 서비스 접근을 예측 가능하게 유지.

## Public Profile

- 공개 프로필을 서버 구조 중심에서 운영 중인 서비스와 엔지니어링 오너십 중심으로 바꿨습니다.
- 현재 방향: 커리어 포지셔닝과 운영 서비스 맥락을 보여주되, 비공개 서버 세부사항은 노출하지 않는 것.

## Quartz Publishing

- 비공개 지식 베이스에서 선택한 콘텐츠를 퍼블리싱하는 레이어로 정리했습니다.
- 현재 방향: 개인 노트에서 공개 가능한 페이지로 이어지는 반복 가능한 흐름 유지.

## Lightweight Web Services

- 작은 정적/PHP 유틸리티를 메인 프로필과 분리된 가벼운 실험으로 정리했습니다.
- 현재 방향: 유용한 프로토타입은 보존하되, 공개 프로필의 핵심 메시지를 흐리지 않게 관리.

## Obsidian Sync

- 동기화 레이어를 계정, 경로, 접근 정보 없이 기능 수준으로만 문서화했습니다.
- 현재 방향: 비공개 지식 저장소의 통제된 동기화와 자동화 접근 지원.

## Ops Lab Documentation

- 서비스별 컨텍스트와 공개 가능한 변경 방향을 `ops-lab` 안에 정리했습니다.
- 현재 방향: 실제 운영 서비스의 맥락은 꾸준히 업데이트하되, 시크릿과 내부 운영 정보는 공개 문서에서 분리.
