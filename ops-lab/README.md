# Ops Lab

[English](./README.en.md)

개인 서버 위에서 제품 개발, AI 보조 워크플로우, 개인 데이터 관리,
지식 관리와 퍼블리싱을 하나의 작은 운영 환경으로 다루고 있습니다.

목표는 많은 서비스를 나열하는 것이 아니라, 제가 실제로 쓰는 생활과
업무 시스템을 AI 중심으로 다시 설계하면서 나만의 AX(AI Transformation)를
실현해보는 것입니다. 사진 보관, 서버 운영, 지식 관리, 커리어 판단,
글쓰기 같은 반복 흐름을 작게 자동화하고, 운영 가능한 형태로 계속
개선합니다.

## 집중하는 문제

- Personal AX
- AI-assisted Operations & Development
- Developer Experience & Workflow Automation
- Knowledge, Career Context & Publishing

## 운영 중인 주요 서비스

### [Public Profile](https://vumy.kr/)

커리어 포지셔닝과 운영 중인 서비스들을 공개 가능한 수준으로 보여주는
프로필 진입점입니다.

### Immich

개인 사진과 영상을 장기 보관하는 실제 미디어 시스템입니다. 저장소
연속성, 데이터 보존, 백업, 서비스 헬스, 복구 가능성을 운영 관점에서
관리합니다.

최근에는 단일 서버에 사진을 쌓아두는 수준을 넘어서, 주 서버 이전,
기존 저장소와 새 저장소의 역할 분리, 백업 검증, 중복 정리 정책까지
포함한 장기 운영 문제로 다루고 있습니다.

### Vodex

LibreChat과 Codex 호환 어댑터를 중심으로 만든 사설 AI 작업 환경입니다.
서버 운영, 문서 작성, 코드 리뷰, 반복 업무 자동화에 AI를 어떻게
안전하게 붙일 수 있는지 실험합니다.

Vodex Console은 이 흐름을 더 직접적인 운영자 콘솔로 확장한 실험입니다.
세션 지속성, 작업 취소, 스트리밍 상태, Telegram 입력 라우팅처럼 실제
운영 중에 필요한 세부 동작을 다룹니다.

### Career AI

지원 자료, 역할 맥락, 포지셔닝을 구조화하는 개인 커리어 워크플로우
프로젝트입니다. 공개 문서에서는 내부 이력서나 지원 자료를 드러내지
않고 워크플로우 수준만 다룹니다.

### Obsidian / Quartz

개인 지식 관리와 선택적 퍼블리싱을 연결하는 파이프라인입니다. 비공개
노트는 보호하면서, 공개 가능한 지식만 반복 가능한 방식으로 내보내는
흐름을 관리합니다.

### Operations Toolkit

운영용 툴박스, 모니터링, 보안 리포트 자동화를 함께 관리합니다. 목적은
큰 관제 시스템을 만드는 것이 아니라, 디스크 압박, 서비스 헬스, 외부
노출, 백업 최신성처럼 개인 서버에서 실제로 놓치기 쉬운 신호를 작고
반복 가능한 점검으로 바꾸는 것입니다.

## 최근에 다루는 운영 고민

- 사진 보관, 지식 관리, 커리어 판단, 글쓰기 같은 개인 워크플로우를 AI 보조 시스템으로 바꾸는 방법
- AI 에이전트가 실제 서버와 데이터를 다룰 때 필요한 권한 경계, 세션 지속성, 취소 가능한 작업 흐름
- 공개 프로필에 올릴 수 있는 AX 실험과 비공개 인프라 정보 사이의 경계
- 작은 서비스를 계속 늘리는 것보다, 상태 확인과 복구 절차가 남는 구조로 정리하는 방법

## 운영 책임

- Docker 기반 배포와 서비스 유지보수
- 웹 진입점과 WAF 설정 관리
- 서비스 헬스 체크, 로그 확인, 장애 후속 조치
- 리소스 모니터링과 용량 계획
- 셀프호스팅 서비스 설정 관리
- 미디어, 데이터베이스, 주요 설정의 백업 계획

## 보여주고 싶은 역량

이 환경은 단순히 컨테이너를 많이 띄워둔 목록이 아니라, 제가 실제로
쓰는 시스템을 AI와 운영 구조로 재설계해가는 개인 AX 기록입니다.

- 개인 생산성 문제를 유지보수 가능한 AI 보조 서비스 경계로 나누는 능력
- 초기 구현 이후에도 사용자-facing 서비스를 계속 다듬는 운영 감각
- 반복 작업을 AI 보조 워크플로우와 자동화로 바꾸는 실행력
- 제품 품질, 운영 안정성, 보안 제약을 함께 고려하는 판단력
- AI를 팀과 개인의 개발 워크플로우 안에 실용적으로 배치하는 관점

## 공개 범위

이 문서는 비공개 설정, 시크릿, 내부 IP, 데이터베이스 인증 정보, 미디어
경로, 백업 경로, 개인 데이터, 사내 세부 정보를 공개하지 않습니다.
